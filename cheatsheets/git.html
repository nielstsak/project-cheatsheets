GIT CONFIG & SETUP

git config --global user.name <name>

Définit le nom de l'auteur utilisé pour tous les commits de l'utilisateur actuel.

git config --global user.email <email>

Définit l'email de l'auteur utilisé pour tous les commits.

git config --global alias.<nom> <commande>

Crée un raccourci clavier. Exemple : alias.glog "log --graph --oneline".

git config --system core.editor <editor>

Définit l'éditeur de texte par défaut pour la machine (ex: vim, nano, code).

git config --global --edit

Ouvre le fichier de configuration global dans l'éditeur pour modification manuelle.

LOGS & ANALYSE

git log -<n>

Limite l'affichage aux n derniers commits. Exemple : git log -5.

git log --oneline

Affiche chaque commit sur une ligne unique (Hash court + Message).

git log --stat

Inclut les statistiques de modification (fichiers changés, lignes +/-) pour chaque commit.

git log -p

Affiche le "diff" complet (le patch) pour chaque commit.

git log --author="<pattern>"

Recherche uniquement les commits d'un auteur spécifique.

git log --grep="<pattern>"

Recherche les commits dont le message contient le motif spécifié.

git log <since>..<until>

Affiche les commits entre deux références (ID, branche, HEAD).

git log <file>

Limite l'historique aux modifications touchant uniquement le fichier spécifié.

git log --graph --decorate

Affiche l'arbre des branches en ASCII (graph) et les noms des refs (decorate).

COMPARAISON & ANNULATION

git diff HEAD

Différence entre le répertoire de travail (non-staged) et le dernier commit.

git diff --cached

Différence entre l'index (staged) et le dernier commit.

git reset <file>

Retire le fichier de l'index (Unstage) mais conserve les modifications locales.

git reset <commit>

Recule la branche actuelle vers <commit>, reset l'index, mais garde le travail local.

git reset --hard

Réinitialise index ET travail au dernier commit. Écrase toutes les modifications locales.

git reset --hard <commit>

Force l'état du projet entier à l'état du <commit> spécifié. Destructif.

git rebase -i <base>

Mode interactif : permet de fusionner, éditer ou supprimer des commits avant le push.

PULL & PUSH (DISTANT)

git pull --rebase <remote>

Récupère la branche distante et rejoue (rebase) vos commits locaux par-dessus.

git push <remote> --force

Force l'envoi même en cas de conflit d'historique. Dangereux (écrase l'historique distant).

git push <remote> --all

Envoie toutes les branches locales vers le dépôt distant.

git push <remote> --tags

Envoie tous les tags locaux vers le serveur (action non automatique par défaut).
