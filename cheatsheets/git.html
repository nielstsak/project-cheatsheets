Initialisation & Config

git init <directory>

Crée un dépôt vide. Si aucun argument n'est fourni, initialise le dossier courant[cite: 3].

git config --global user.name <name>

Définit le nom de l'auteur utilisé pour tous les commits de l'utilisateur actuel[cite: 3, 10].

git config --global alias.<alias> <cmd>

Crée un raccourci. Exemple : alias.glog "log --graph"[cite: 10].

git clone <repo>

Clone le dépôt distant sur la machine locale via HTTP ou SSH[cite: 3].

Gestion des Changements

git status

Affiche les fichiers indexés (staged), modifiés (unstaged) et non-suivis (untracked)[cite: 3].

git add <directory>

Ajoute les modifications du répertoire à l'index pour le prochain commit[cite: 3].

git commit -m "<message>"

Valide le snapshot indexé avec un message descriptif sans ouvrir l'éditeur[cite: 3].

git diff --staged

Affiche les différences entre les fichiers indexés et le dernier commit[cite: 13].

Historique & Logs

git log --graph --oneline --decorate

Affiche l'arbre des commits sous forme de graphe ASCII compact avec les refs[cite: 10].

git log -p <file>

Affiche l'historique complet des diffs pour un fichier spécifique[cite: 10].

git reflog

Journal de sécurité affichant tous les mouvements de HEAD (utile pour récupération après erreur)[cite: 3].

Branches & Synchro Distante

git checkout -b <branch>

Crée une nouvelle branche et bascule immédiatement dessus[cite: 3].

git merge <branch>

Fusionne la branche spécifiée dans la branche courante[cite: 3].

git fetch <remote>

Récupère les objets distants sans modifier le répertoire de travail local[cite: 3].

git push <remote> --tags

Envoie tous les tags locaux vers le dépôt distant (non inclus par défaut)[cite: 38].

Réécriture & Correction

git commit --amend

Combine les changements indexés avec le commit précédent (ou édite le message)[cite: 3].

git rebase -i <base>

Rebase interactif pour nettoyer, fusionner ou réordonner les commits avant push[cite: 25, 26].

git reset --hard <commit>

Réinitialise l'index ET le répertoire de travail pour correspondre au commit cible. Destructif[cite: 17, 18].

git revert <commit>

Crée un commit inverse pour annuler les effets d'un commit précédent (Safe)[cite: 3].
