GIT CONFIG & SETUP

git config --global user.name <name>

Définit le nom de l'auteur utilisé pour tous les commits de l'utilisateur actuel[cite: 3, 10].

git config --global user.email <email>

Définit l'email de l'auteur pour tous les commits.

git config --global alias.<alias> <cmd>

Crée un raccourci. Ex: alias.glog "log --graph --oneline".

git config --system core.editor <editor>

Définit l'éditeur de texte pour les commandes (ex: vi, nano) pour tous les utilisateurs.

git config --global --edit

Ouvre le fichier de configuration global dans un éditeur de texte pour modification manuelle.

LOGS & HISTORIQUE

git log -<limit>

Limite le nombre de commits affichés. Ex: git log -5.

git log --oneline

Condense chaque commit sur une seule ligne (ID + Message).

git log --stat

Affiche quels fichiers ont été modifiés et le nombre relatif de lignes ajoutées/supprimées.

git log -p

Affiche le diff complet (patch) pour chaque commit.

git log --author="<pattern>"

Recherche les commits par un auteur spécifique.

git log --grep="<pattern>"

Recherche les commits dont le message correspond au motif.

git log <since>..<until>

Affiche les commits entre deux références (ID, branch, HEAD).

git log --graph --decorate

Dessine un graphe textuel des branches et ajoute les noms des branches/tags.

DIFF & RESET (UNDOING)

git diff HEAD

Affiche la différence entre le répertoire de travail et le dernier commit[cite: 11, 13].

git diff --cached

Affiche la différence entre l'index (staged) et le dernier commit[cite: 12].

git reset <file>

Retire le fichier de l'index (Unstage) mais garde le répertoire de travail inchangé[cite: 16].

git reset <commit>

Recule le pointeur de branche vers <commit>, reset l'index, mais garde le répertoire de travail[cite: 19, 20].

git reset --hard <commit>

Réinitialise l'index ET le répertoire de travail. Supprime tous les changements non committés[cite: 17, 18, 21].

REMOTE, PULL & PUSH

git rebase -i <base>

Rebase interactif. Lance l'éditeur pour gérer le transfert de chaque commit (squash, fixup)[cite: 25, 26].

git pull --rebase <remote>

Récupère la branche distante et rebase la branche locale dessus (au lieu de merger)[cite: 28, 29].

git push <remote> --force

Force le push même si cela entraîne un "non-fast-forward merge". Dangereux[cite: 33, 34].

git push <remote> --all

Envoie toutes les branches locales vers le dépôt distant spécifié[cite: 36, 37].

git push <remote> --tags

Envoie tous les tags locaux vers le dépôt distant (ne se fait pas par défaut)[cite: 38, 40].
